Detectar endereço IP externo do sistema usando DNS
Esta opção usa o comando dig para consultar um servidor DNS e detectar o endereço IP externo do sistema. Por exemplo, você pode definir este campo como myip.opendns.com resolver1.opendns.com (um nome de consulta e um servidor de nomes separados por espaços). Isso evita a dependência do método baseado em HTTP e recupera o endereço IP diretamente do servidor DNS.

No entanto, o método DNS deve ser usado com cautela se o sistema estiver atrás de um NAT, proxy ou VPN, pois o IP detectado pode não refletir o IP público real da máquina. Em vez disso, ele pode mostrar o IP externo do gateway NAT, proxy ou ponto de extremidade VPN.
