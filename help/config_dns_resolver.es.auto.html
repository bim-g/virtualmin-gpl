Detectar la dirección IP externa del sistema mediante DNS
Esta opción utiliza el comando dig para consultar un servidor DNS y detectar la dirección IP externa del sistema. Por ejemplo, puede configurar este campo como myip.opendns.com resolver1.opendns.com (un nombre de consulta y un servidor de nombres separados por espacios). Esto evita depender del método HTTP y recupera la dirección IP directamente del servidor DNS.

Sin embargo, el método DNS debe usarse con precaución si el sistema está protegido por un NAT, un proxy o una VPN, ya que la IP detectada podría no reflejar la IP pública real del equipo. En su lugar, podría mostrar la IP externa de la puerta de enlace NAT, el proxy o el punto final de la VPN.
