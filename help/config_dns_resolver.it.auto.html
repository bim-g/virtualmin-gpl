Rileva l'indirizzo IP esterno del sistema tramite DNS
Questa opzione utilizza il comando dig per interrogare un server DNS e rilevare l'indirizzo IP esterno del sistema. Ad esempio, è possibile impostare questo campo su myip.opendns.com resolver1.opendns.com (un nome di query e un nameserver separati da spazi). In questo modo si evita di affidarsi a un metodo basato su HTTP e l'indirizzo IP viene recuperato direttamente dal server DNS.

Tuttavia, il metodo DNS deve essere utilizzato con cautela se il sistema è protetto da NAT, proxy o VPN, poiché l'IP rilevato potrebbe non riflettere l'IP pubblico effettivo della macchina. Potrebbe invece mostrare l'IP esterno del gateway NAT, del proxy o dell'endpoint VPN.
